## 검증 이력

| 차수 | 방법 | 결과 |
|------|------|------|
| 1차 | 기능 분석 문서 vs 웹 코드 대조 | 제거 6건, 추가 22건 도출 |
| 2차 | Android 소스코드 (`desktop/workspace/aos`) 교차 검증 | 추가 항목 전부 확인, 제거 항목 유지 |
| 3차 (최종) | **실제 앱 스크린샷 검증** | 서버 주도 UI로 코드에 미반영된 섹션 다수 발견 → **제거 대상 6건 → 2건으로 축소**, 4건 유지 복원 |

### 3차 검증에서 뒤집힌 항목

> Android 앱은 서버 주도(Server-Driven) UI 구조로, 홈 화면의 많은 섹션이 서버에서 동적으로 구성됨.
> 소스코드에는 범용 렌더러만 존재하고, 실제 표시되는 콘텐츠(매거진, 기획전, 프로모카드, 지역 추천 등)는 서버 응답에 의해 결정됨.
> 따라서 코드 분석만으로는 이 섹션들의 존재를 확인할 수 없었으며, 실제 앱 스크린샷을 통해 최종 확인함.

| 항목 | 1~2차 판정 | 3차 최종 판정 | 근거 |
|------|-----------|-------------|------|
| **MagazineSection** | 제거 | ✅ **유지** | 앱에 "꿀스테이 여행 매거진" 섹션 + "매거진 더보기" 존재 확인 |
| **PromoCards** | 제거 | ✅ **유지** | 앱에 "무제한 혜택/국내 최저가/첫예약 선물" 3카드 동일 구성 존재 확인 |
| **FeatureSection** | 제거 | ✅ **유지** | 앱에 "기획전" 섹션 배너 존재 확인, 카테고리 그리드에도 "기획전" 포함 |
| **RegionRecommendations** | 변형 (숙소 목록으로 교체) | ✅ **유지** | 앱에 "이런 숙소는 어떠세요?" 지역 탭 + 숙소 그리드 존재 확인 |

### 2.1 HeroSection 내 꿀딜 캐러셀 — 제거

| 항목 | 내용 |
|------|------|
| **파일** | `src/domains/home/components/HeroSection.tsx` 내 꿀딜 영역 |

| **조치** | HeroSection 자체는 유지, 검색바 중심으로 단순화. 꿀딜 관련 코드/데이터만 제거 |
| **관련 Mock 데이터** | `src/domains/home/data/mock.ts` 내 `honeyDeals` 제거 |

### 2.2 Footer/MobileNav 불필요 링크 — 제거

| 대상 | 위치 | 제거 이유 |

| **Footer 파트너 섹션** | `src/components/layout/Footer.tsx` | `/partner`, `/partner/register`, `/partner/ads` — Android에 없는 B2B 기능, 페이지 미구현 |

## 5. 추가 구현 — 즉시 필요 (🔴)

> 핵심 사용자 플로우가 끊기는 항목. 링크는 존재하나 도착 페이지가 없어 404 발생.

### 5.1 예약 상세 페이지

- **라우트**: `/bookings/[id]`
- **진입점**: 예약 내역 목록에서 카드 탭
- **Android 클래스**: `BookingListDetailActivity.kt`
- **구성**:

| 영역 | 내용 |
|------|------|
| 숙소/객실 정보 | 숙소명, 이용유형(숙박/대실 + 박수), 객실명, "숙소 보기" 버튼 |
| 예약 정보 | 예약 상태 뱃지, 예약번호, 교통수단, 체크인/체크아웃 시간, 예약자명(마스킹), 전화(마스킹), 안심번호, 예약 일시 |
| 결제 정보 | 결제수단, 판매가, 예약할인, 쿠폰1 할인, 쿠폰2 할인, 마일리지 할인, 총 결제금액 |
| 길찾기 | 지도 프리뷰, 주소, 주소복사 버튼 |
| 액션 | 리뷰 작성/보기, 공유, 삭제, 예약 취소, 재예약 (상태별 분기) |
| 푸터 | 법적 고지, 고객센터 전화번호 |

- **액션 상태별 분기**:
  - 예약완료: 예약 취소 버튼 활성
  - 이용완료 + 리뷰 미작성: "리뷰 작성" 버튼
  - 이용완료 + 리뷰 작성 완료: "리뷰 보기" 버튼
  - 이용완료: "재예약" 버튼 → 숙소 상세 이동
  - 공통: 공유, 삭제

### 5.2 비밀번호 찾기 페이지

- **라우트**: `/forgot-password`
- **진입점**: 로그인 페이지 "비밀번호 찾기" 링크
- **Android 클래스**: `ResetPasswordInputEmailActivity.kt` → `CertificationActivity`
- **구성**:

| 단계 | 내용 |
|------|------|
| Step 1 | 이메일 입력 → 서버에서 인증수단 조회 |
| Step 2 | 이메일 인증 / SMS 인증 택 1 (마스킹된 정보 표시) |
| Step 3 (SMS) | 인증코드 입력 + 카운트다운 타이머 |
