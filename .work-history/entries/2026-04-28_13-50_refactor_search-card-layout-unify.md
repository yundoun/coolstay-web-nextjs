# 검색 결과 카드 레이아웃 높이 통일

## 변경 사항
- AccommodationCard 가격 영역을 항상 2열 grid로 렌더링 (min-h-[103px])
- 하단 혜택 바를 항상 렌더링 (고정 1줄, h-[30px]), 플래그 최대 3개 + 마일리지 우측 고정
- 단일 가격 카드도 2열 구조 내에서 표시하여 높이 차이 제거
- PriceColumn에 대실 이용시간, 숙박 체크인, 잔여 객실 수 표시 추가

## 이유
- 혜택 플래그 수에 따라 카드 높이가 280px ~ 315px로 불일치
- 2열 그리드에서 나란히 놓인 카드의 높이 차이가 시각적으로 어색
- 앱 실제 디자인 기준으로 통일 (고정 1줄 하단 바 + 2열 가격)
